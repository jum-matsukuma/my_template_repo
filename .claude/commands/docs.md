# Generate Documentation

Create comprehensive documentation for code, APIs, and project components.

## Usage

- `/docs` - Generate documentation for modified files
- `/docs [file-path]` - Generate docs for specific file
- `/docs api` - Generate API documentation
- `/docs $ARGUMENTS` - Generate docs based on arguments

## What This Command Does

This command creates clear, comprehensive documentation including:

1. **Function Documentation** - JSDoc, docstrings, and inline comments
2. **API Documentation** - Endpoint descriptions and examples
3. **Usage Examples** - Practical code examples
4. **Architecture Overview** - High-level design documentation
5. **Installation/Setup** - Getting started guides
6. **Troubleshooting** - Common issues and solutions

Documentation follows the project's standards and includes practical examples for developers.
