# Generate Tests

Create comprehensive test suites for specified code or components.

## Usage

- `/test` - Generate tests for recently modified files
- `/test [file-path]` - Generate tests for specific file
- `/test $ARGUMENTS` - Generate tests based on arguments

## What This Command Does

This command creates robust test suites including:

1. **Unit Tests** - Test individual functions and components
2. **Integration Tests** - Test component interactions
3. **Edge Cases** - Boundary conditions and error scenarios
4. **Mock Patterns** - Proper mocking of dependencies
5. **Test Organization** - Clear test structure and naming

Tests follow best practices for the project's testing framework and include clear descriptions, arrange-act-assert patterns, and good coverage of critical functionality.
