# Review Code

Perform a comprehensive code review of the current project or specified files.

## Usage

- `/review` - Review all modified files
- `/review [file-path]` - Review specific file
- `/review $ARGUMENTS` - Review files matching arguments

## What This Command Does

This command triggers a thorough code review covering:

1. **Code Quality** - Readability, maintainability, best practices
2. **Performance** - Potential bottlenecks and optimization opportunities  
3. **Security** - Common vulnerabilities and secure coding practices
4. **Testing** - Test coverage and quality assessment
5. **Architecture** - Design patterns and scalability considerations
6. **Documentation** - Code comments and clarity

The review will provide specific, actionable feedback with examples and prioritize the most impactful improvements.
