# Cleanup Code

未使用コード、不要なインポート、デッドコードを検出・除去する。

## Usage

- `/cleanup` - プロジェクト全体をスキャン
- `/cleanup [file-path]` - 指定ファイルをクリーンアップ
- `/cleanup $ARGUMENTS` - 指定された範囲をクリーンアップ

## What This Command Does

以下の項目を検出して整理する:

### 1. 未使用コードの検出
- 未使用のインポート文
- 未使用の変数・関数・クラス
- 到達不能コード（unreachable code）
- コメントアウトされた古いコード

### 2. コードの整理
- 検出した不要コードを削除
- インポート文の整理・ソート
- 不要な空行やトレイリングスペースの除去

### 3. 設定ファイルの整理
- package.json の未使用dependenciesを報告
- 使われていない設定ファイルの検出

### 4. 確認と報告
- 変更内容をファイルごとにサマリ表示
- 削除した項目の一覧を報告
- 判断が難しい項目はユーザーに確認

## Notes

- 外部から参照される可能性があるエクスポートは保守的に判断
- テストファイルは対象外（テスト用のモックやヘルパーは残す）
- 変更前に `git status` で未コミットの変更がないことを確認
